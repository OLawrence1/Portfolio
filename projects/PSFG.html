Overview

For my Programming Systems for Games module, I developed an AI battle simulation in Unity where two teams of autonomous agents compete inside a procedurally generated maze until only one team remains.

The project combined several gameplay systems including procedural maze generation, graph-based pathfinding, finite state machine AI, and an online leaderboard to create a replayable simulation with no direct player control.

My Contribution

This was an individual project where I designed and implemented every gameplay system from scratch, integrating multiple independent systems into a single AI-driven experience.

  • Implemented procedural maze generation using a randomised Depth First Search algorithm.
  • Created a custom grid-based navigation graph and A* pathfinding system.
  • Developed finite state machine AI controlling navigation, combat and decision making.
  • Integrated an online leaderboard using the Dreamlo web service API.
  • Combined all systems into a fully autonomous battle simulation.

Technical Breakdown

The procedural generation system creates a unique maze at runtime using a randomised Depth First Search algorithm. This guarantees a fully connected environment while ensuring every simulation takes place in a different layout, increasing replayability and preventing predictable encounters.

AI navigation is handled through a custom grid-based navigation graph generated after the maze is built. Agents use the A* pathfinding algorithm to calculate efficient routes through the environment while avoiding walls and obstacles.

Agent behaviour is controlled through a Finite State Machine (FSM) consisting of states such as patrolling, chasing enemies, attacking, and collecting weapons. State transitions are driven by information gathered from the environment and current gameplay conditions, allowing agents to react dynamically during combat.

To encourage replayability beyond the simulation itself, I integrated an online leaderboard using the Dreamlo API. Player scores can be uploaded, downloaded, sorted and displayed in-game, allowing different simulation results to be compared between sessions.

Challenges

One of the largest technical challenges involved synchronising the procedural generation pipeline. Initially, the navigation graph was generated before the maze had finished creating, resulting in nodes appearing inside walls and causing invalid paths.

This was resolved by restructuring the generation sequence using coroutines, ensuring the maze completed first, followed by navigation graph creation and finally AI spawning. This dependency ordering eliminated invalid navigation data and allowed all systems to integrate correctly.

Another issue arose when pathfinding targets were recalculated every frame, causing agents to repeatedly restart path searches and become stuck. Optimising updates so destinations were only recalculated when necessary significantly improved movement performance and stability.

Outcome

The final project successfully combined procedural generation, pathfinding, AI behaviour systems and live services into a cohesive simulation where autonomous agents dynamically explore, fight and adapt within unique environments every playthrough.

The project strengthened my understanding of AI architecture, navigation systems, procedural generation techniques and the complexities of integrating multiple gameplay systems into a scalable framework.
